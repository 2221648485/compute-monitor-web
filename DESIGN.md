# Compute Monitor Design

本项目采用 NVIDIA 风格作为控制台视觉基准，目标是工程感、清晰、克制，而不是营销感或装饰感。

## 视觉原则

- 主色只使用 NVIDIA Green `#76b900`，用于主按钮、激活态、关键状态和角标。
- 页面主体使用黑、白、灰：`#000000`、`#ffffff`、`#f7f7f7`、`#cccccc`、`#757575`。
- 所有卡片、输入框、按钮保持 2px 小圆角，不使用大圆角、渐变背景、彩色光斑或柔和阴影。
- 信息布局偏技术文档和运维控制台：标题清晰，表格密集但留足行高，字段用细线分隔。
- 中文作为主要界面语言，Kubernetes、Prometheus、GPU、Pod、Deployment、RBAC 等技术名词保留英文。

## 组件约束

- 主按钮：绿色背景、黑色文字、44px 高度、2px 圆角。
- 次按钮：白底、灰色细边框、黑色文字。
- 表格：英文表头，表头灰底，单元格用细线分隔，避免状态胶囊堆叠造成视觉噪音。
- 卡片：白底、1px 灰色边框、右上角 12px 绿色角标。
- 列表承载大量数据时必须使用分页或内部滚动，不能让页面无限拉长。

## 页面规则

- 集群纳管页要突出集群名称，集群 ID 是辅助信息，不能把名称和 ID 挤在同一个小块里。
- 节点/GPU 页的 Nodes、GPU 使用率和节点清单都必须有内部滚动容器。
- 工作负载页使用服务端搜索和分页。搜索 label 与输入框保持同一行，避免“搜索 Pod”和输入框上下分离。
- 所有错误反馈通过顶部 Toast 自动消失，不在页面中长期驻留。
